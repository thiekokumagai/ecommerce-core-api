# Plan 06-01: Banco de Dados e Módulo API de Configurações (Backend)

## Objective
Implementar a persistência do banco de dados (Prisma) e a API NestJS (`ecommerce-core-api`) para o módulo `StoreSettings` seguindo o padrão da Clean Architecture Modular.

## Tasks

<task id="1" title="Adicionar modelo StoreSettings no Prisma Schema">
<read_first>
- prisma/schema.prisma
- .planning/phases/06-implementa-o-do-m-dulo-de-configura-es-storesettings/CONTEXT.md
</read_first>

<action>
Adicionar o modelo `StoreSettings` no final do arquivo `prisma/schema.prisma`:
- Usar map `@@map("store_settings")`
- Definir campos para nome da loja, urls de logo, favicon e banners (até 7), telefone, instagram, endereço completo com CEP, switches de Pix e chaves, switches de pagamentos presenciais e campo JSON para regras de parcelamento e desconto.

</task>

1. `get-settings.use-case.ts`:
   - Buscar configurações usando `settingsRepository.get()`.
